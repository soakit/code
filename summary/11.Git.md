# rebase sample

## 1. 初始化git

```shell
git init
```

## 2. master分支操作

```shell
# master.txt添加内容1
vi master.txt
### master.txt
### 1
git add .
git commit -m "master 1"

# 创建feature分支
git checkout -b feature
git checkout master

# master.txt添加内容 2
vi master.txt
### master.txt
### 1
### 2
git add .
git commit -m "master 2"

# master.txt添加内容 3
vi master.txt
### master.txt
### 1
### 2
### 3
git add .
git commit -m "master 3"
```

## 3. feature分支操作

```shell
git checkout feature
vi feature.txt
git add .
git commit -m "feature init"

# master.txt添加内容 conflict 1
vi master.txt
### master.txt
### 1
### conflict 1
git add .
git commit -m "conflict 1"

# master.txt添加内容 conflict 2
vi master.txt
### master.txt
### 1
### conflict 1
### conflict 2
git add .
git commit -m "conflict 2"

vi feature.txt
### feature.txt
### 1
git add .
git commit -m "feature commit"
```

## 4. rebase操作

```shell
# feature分支
git rebase -i master
# 解决冲突
```
